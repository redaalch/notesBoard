# Security Policy

## Private Vulnerability Reporting

Report sensitive security issues privately to `security@notesboard.xyz`.

Please do not open a public GitHub issue for exploitable vulnerabilities.

Include the following when possible:

- A short description of the issue and affected area
- Reproduction steps or a proof of concept
- Expected impact and any known mitigations
- Your preferred contact details for follow-up

## Response Expectations

- Initial acknowledgement within 3 business days
- Status updates at least every 7 business days while the report is active
- Coordinated disclosure after a fix or mitigation is available

## Supported Scope

Security fixes are prioritized for the latest code on `main`.
